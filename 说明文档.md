# 医疗助手流式消息对接说明

本文档帮助前端开发接入 `/assistant/chat/stream` 流式接口，实时展示医疗助手工作流的执行进度与最终回复。

## 1. 请求说明

- **接口地址**：`POST /assistant/chat/stream`
- **请求头**：
  - `Content-Type: application/json`
  - `Accept: text/event-stream`
- **请求体示例**：

```json
{
  "uuid": "可选，会话唯一标识，没有则留空由后端生成",
  "message": "用户输入的最新一条消息"
}
```

> 当 `uuid` 为空或不存在时，后端会创建新的会话并返回新的 `uuid`；如果传入已有会话的 `uuid`，服务会在原对话上下文中继续工作流。

## 2. SSE 连接示例

可以使用浏览器原生 `EventSource`，也可以使用 `fetch-event-source` 等封装库。示例如下：

```ts
import { fetchEventSource } from '@microsoft/fetch-event-source';

fetchEventSource('/assistant/chat/stream', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    Accept: 'text/event-stream'
  },
  body: JSON.stringify({ message: userInput, uuid: currentSessionUuid }),
  onopen() {
    console.log('SSE 已连接');
  },
  onmessage(event) {
    if (!event.data) {
      return;
    }
    const payload = JSON.parse(event.data);
    handleStreamChunk(payload);
  },
  onerror(err) {
    console.error('SSE 连接错误', err);
  },
  onclose() {
    console.log('SSE 连接已关闭');
  }
});
```

## 3. 返回数据结构

每一条 SSE `data:` 消息都是一个 `StreamChatResponse` 对象，字段说明如下：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `uuid` | `string` | 会话唯一标识，前端需缓存起来，用于后续请求复用同一会话 |
| `messageUuid` | `string` | 助手回复消息的唯一标识，仅在流程最终完成时返回，用于前端记录消息或做幂等处理 |
| `responseId` | `string` | 预留的流式响应 ID（当前未使用，可忽略） |
| `stage` | `string` | 当前工作流阶段编码（见下文“阶段枚举”） |
| `stageMessage` | `string` | 阶段对应的说明文案，用于 UI 提示（如“正在意图识别”） |
| `toolName` | `string` | 当阶段涉及工具调用时返回的 Spring AI 工具名称，例如 `getCurrentTime` |
| `toolMessage` | `string` | 工具执行的结果或摘要信息，可用于展示或写入日志 |
| `content` | `string` | 医疗助手生成的文本分片，前端按顺序拼接即可形成完整回答 |
| `finished` | `boolean` | `true` 表示本次流式响应结束（成功或失败），可以关闭进度提示 |

重要提示：

- **心跳消息**：当 `stage` 为 `heartbeat` 时表示后端正在保活 SSE 连接，可直接忽略显示。
- **阶段消息**（没有 `content`）用于告知用户当前的工作流步骤，可直接渲染在“进度提示”区域。
- **内容分片**（有 `content`）用于滚动输出最终答案的正文，`stage` 会固定为 `response_stream`。
- **工具调用**会发送两类消息：
  - `tool_invoke`：工具开始执行，`toolMessage` 给出上下文说明。
  - `tool_result`：工具执行结果，`toolMessage` 为结果摘要。
- 当 `finished = true` 时，如果 `stage` 为 `completed` 表示任务成功完成；若为 `failed` 表示本轮对话失败（此时 `content` 一般是一段兜底文案）。

## 4. 阶段枚举（`stage`）

`ChatStageEnum` 中的阶段码会在流式过程中依次出现，前端可根据需要做文案映射或图标展示：

| 阶段码 | 说明 |
| --- | --- |
| `received` | 已接收用户消息，准备启动工作流 |
| `workflow_start` | 工作流初始化完成 |
| `heartbeat` | 心跳保活消息（前端可忽略展示，仅用于维持连接） |
| `intent_analysis` | 正在进行意图识别 |
| `route_medicine` | 进入药品咨询分支 |
| `route_consult` | 进入健康咨询分支 |
| `route_other` | 进入其他问题处理分支 |
| `tool_invoke` | 正在调用某个工具（如时间、用户信息等） |
| `tool_result` | 工具调用返回结果 |
| `response_stream` | 正在流式返回助手的最终回答 |
| `completed` | 流程成功结束，`messageUuid` 可用于前端消息列表入库 |
| `failed` | 流程发生异常，通常会有兜底回复同时返回 |

## 5. 前端展示建议

1. **进度提示**：监听 `stage` + `stageMessage`，在对话顶部或侧边展示当前执行节点。
2. **工具高亮**：当 `stage` 为 `tool_invoke` / `tool_result` 时，可在日志面板中显示工具名称与结果，帮助用户理解工作流行为。
3. **正文拼接**：当 `stage` 为 `response_stream` 时，将 `content` 追加到当前回复文本；`finished=true` 后视为本轮回复完成。
4. **失败处理**：若最后一次 `finished=true` 的阶段为 `failed`，使用返回的 `content` 或 `stageMessage` 提示用户，可提供重新尝试按钮。

## 6. 断线重连 & 取消

- 后端会每 10 秒发送一次心跳消息保障连接，如前端在限定时间内未收到任何数据，可提示用户检查网络。
- SSE 连接断开时，前端可根据业务策略选择自动重连；若用户手动停止，可直接调用 `controller.abort()`（使用 Fetch API）或关闭 `EventSource`。
- 后端在检测到客户端取消后会停止继续推送消息，避免额外资源消耗。

如需更多字段或阶段扩展，请与后端同学同步 `StreamChatResponse` 或 `ChatStageEnum` 的字段更新。
